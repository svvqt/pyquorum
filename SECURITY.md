# Security Policy

## Supported Versions
Please use the latest version of the package to ensure that all known vulnerabilities are addressed

## Reporting a Vulnerability
If you have found vulnerability in the Rust core or Python API code please report it responsibly

Please do not report security issues via the public GitHub issue tracker. Instead, use the GitHub Security tab to submit a private report

What report needs to contain:
- description of issue;
- steps to reproduce;
- environment (OS, python ver, package ver).

## What to expect after reporting
I will acknowledge receipt of your report within 48 hours. If the vulnerability is confirmed, i will work on a fix and release a security patch as soon as possible
